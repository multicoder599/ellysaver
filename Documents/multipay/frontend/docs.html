API Documentation

Everything you need to integrate M-Pesa STK Push payments into your application.

shield Authentication

All API requests (except auth endpoints) require an API key in the x-api-key header.

x-api-key: mp_live_abc123def456...

Generate your API key from the Dashboard. Each key is shown only once at creation — store it securely.

send Initiate STK Push

POST /api/stkpush

Triggers an M-Pesa STK Push (Lipa na M-Pesa Online) to the customer's phone via Megapay.

Request Body

{ "phone": "254712345678", "amount": 100, "accountReference": "Order123", "description": "Payment for Order123" }

Response

{ "success": true, "message": "STK Push sent to phone.", "transactionId": "6579a1b2c3d4e5f6a7b8c9d0", "refId": "MP1234567890", "status": "pending", "phone": "254712345678", "amount": 100, "pollUrl": "/api/transactions/6579a1b2c3d4e5f6a7b8c9d0/status" }

refresh Check Transaction Status

GET /api/transactions/:id/status

Poll the status of a pending STK Push transaction using the transaction ID.

curl -X GET https://yourdomain.com/api/transactions/6579a1b2c3d4e5f6a7b8c9d0/status \ -H "x-api-key: YOUR_API_KEY"

webhook Webhooks

When a customer completes or cancels the M-Pesa payment, Megapay sends a callback to /api/webhook/megapay. Our platform processes it and updates your transaction status and wallet balance automatically.

Webhook Payload (Success)

{ "ResponseCode": 0, "ResultDesc": "The service request is processed successfully.", "TransactionAmount": 100, "TransactionReceipt": "SGA7X12345", "Msisdn": "254712345678", "reference": "MP1234567890" }
